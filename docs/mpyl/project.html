mpyl.project

Datamodel representation of project specific configuration as specified in the deployment/project.yml. It defines how the source code to which it relates "wants" to be built / tested / deployed.

Run tests and checks

To run linting (pylint), type checking (mypy) and testing (pytest) in one go, run: 

pipenv run validate

Troubleshooting Python setup

  1. Check if you're in the correct venv To check this, run first:

    pipenv shell

    Then check if the correct virtual environment (named pympl) is launched.

  2. Check your bashrc (or zshrc) if you have any overrides of environmental variables like PIPENV_PIPFILE. If so, remove those, source your bash config and try Step 1. again

  3. To see if everything is running as intended, execute

    pipenv run test

    which should now succeed.

 33@dataclass(frozen=True)
 34class TargetSpecificProperty(Generic[T]):
 35    pr: Optional[T]  # pylint: disable=invalid-name
 36    test: Optional[T]
 37    acceptance: Optional[T]
 38    production: Optional[T]
 39    all: Optional[T]
 40
 41    def get_value(self, target: Target):
 42        if self.all:
 43            return self.all
 44        if target == Target.PULL_REQUEST:
 45            return self.pr
 46        if target == Target.PULL_REQUEST_BASE:
 47            return self.test
 48        if target == Target.ACCEPTANCE:
 49            return self.acceptance
 50        if target == Target.PRODUCTION:
 51            return self.production
 52        return None

 54    @staticmethod

 56        return TargetSpecificProperty(pr=values.get('pr'), test=values.get('test'), acceptance=values.get('acceptance'),

278def load_project(root_dir, project_path: str, strict: bool = True) -> Project:
279    """
280    Load a `project.yml` to `Project` data class
281    :param root_dir: root source directory
282    :param project_path: relative path from `root_dir` to the `project.yml`
283    :param strict: indicates whether the schema should be validated
284    :return: `Project` data class
285    """
286    with open(f'{root_dir}/{project_path}', encoding='utf-8') as file:
287        try:
288            yaml = YAML()
289            yaml_values = yaml.load(file)
290            template = pkgutil.get_data(__name__, "schema/project.schema.yml")
291            if strict and template:
292                schema = yaml.load(template.decode('utf-8'))
293                jsonschema.validate(yaml_values, schema)
294
295            return Project.from_yaml(yaml_values, project_path)
296        except jsonschema.exceptions.ValidationError as exc:
297            logging.warning(f'{project_path} does not comply with schema: {exc.message}')
298            raise
299        except TypeError:
300            traceback.print_exc()
301            logging.warning('Type error', exc_info=True)
302            raise
303        except Exception:
304            logging.warning(f'Failed to load {project_path}', exc_info=True)
305            raise
